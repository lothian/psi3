Checklist of necessary components

The following is a default list of libraries needed to compile CINTS (all of these are a part of PSI3.0):

  • LIBCIOMR

  • LIBIPV1

  • LIBPSIO

  • LIBIWL

  • LIBQT

  • LIBFILE30

  • LIBINT

  • LIBDERIV

For certain configurations, CINTS may also require LIBR12 from PSI3.0. It is recommended that a vendor-provided or generic BLAS library is used to achieve better performance in the integral-direct transformation stage (see below how to configure CINTS to use BLAS library routines).

CINTS is mostly useless as a standalone program: it has to work in conjunctions with other programs. However, I am not going to describe how to compile the programs used in conjunctions with PSI3.0. Most likely you want to check out and compile the following binaries which work closely in conjunction with CINTS:

  • INPUT3.0

  • CSCF3.0

  • TRANSQT

  • MP2R12

Please refer to respective documentation pages for more information.

Compiling LIBINT, LIBDERIV, and LIBR12 libraries

This process has not been automated yet. Follow the steps:

  • Compile and install BUILD_LIBINT program.

  • Go to src/lib/libint, there should be no source files inside. These libraries are generated by computer codes, so the compilation process is slightly more complicated than usual. To become an expert in how to compile the library you need to read README. However, assuming that you're compiling it for the first time, directory /tmp1/$(USER) exists, you have write permissions to it, and /tmp1 has more than 30 MB of free space left, you can safely proceed as follows: edit the input.dat file and set the new_am option to twice the maximum orbital quantum number of basis functions you want to be able to handle with CINTS. It does not have to be the same as MAXANGMOM value in file30_params.h . Type 'make'. It is going to be awhile.. By default, CINTS is going to quit if the LIBINT library it was linked to can't handle the given basis set. I generally do not recommend to go higher than i-functions unless you know what you're doing. If you want to run test cases - new_am = 8 (g-functions) is more than enough. For production level work -- just follow your needs. Remember that LIBINT can become pretty big. For example, the version capable of handling i-functions is going to be about 15-20 MB big. Not too bad though, because even my laptop has 1 GB in /tmp1.

  • After this, to compile LIBDERIV (LIBR12) - compile and install BUILD_LIBDERIV (BUILD_LIBR12), and then compile LIBDERIV (LIBR12).

Configuration

  • Depending on your system environment and what capabilities of CINTS you want to enable, you might have to edit Makefile first. Add subdirectories which contain code you want to include to SUBDIRS and edit CLIBS accordingly. Here's the current list of directories and the required additions to CLIBS:

  • If a BLAS library is available - add it to CLIBS in Makefile, set USE_BLAS in defines.h to 1, and make sure the $(BIN) target uses $(F77) as a linker rather than $(CC). If no BLAS library is available - USE_BLAS should be set to 0.

  • If you're trying to compile CINTS on a system other than AIX/RS6k or Linux/x86 - you may want to edit compiler flags in ../../MakeVars to get the best performance.

Compilation

Go to src/bin/cints, type 'make install'. Voila!
